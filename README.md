# 🃏 드래프트 포커 (Draft Poker)

보드게임의 **드래프트 메커니즘**과 **포커 베팅**을 결합한 오리지널 카드 게임.
1단계(로컬 + AI 봇) 구현 — 2단계에서 Firebase 실시간 멀티플레이 추가 예정.

## 룰 v1.0

- **인원**: 3~6인, 표준 52장 덱
- **손패**: 인원수+1장 (3인은 5장 보장) / 시작 뭉치: 손패+1장
- **드래프트**: 뭉치에서 1장을 고르고 나머지를 옆 사람에게 패스. **패스 방향은 매판 랜덤**
- **마지막 픽**: 2장 중 1장 선택, 남은 1장은 비공개 폐기
- **베팅**: 픽을 3구간으로 나눠 구간마다 한게임식 제한 베팅
  - 오픈(아직 베팅 없음): **다이/삥/쿼터/하프** — 체크 없음, 무조건 띄워야 함
  - 베팅을 받음: **다이/콜/따당/쿼터/하프/풀**
  - **콜 후 리레이즈 금지** (한국 룰: 콜을 달면 그 라운드는 콜/다이만)
  - 올인 버튼 없음 — 칩이 모자랄 때만 자연 발생
  - 턴 제한: 베팅 15초(초과 시 자동 다이) / 드래프트 픽 20초(초과 시 무작위 자동 픽)
- **폴드**: 들고 있던 뭉치를 통째로 비공개 폐기 — 드래프트·베팅 모두 이탈
- **쇼다운**: 손패에서 베스트 5장 족보 대결 (홀덤 표준 족보, 사이드팟 지원)
- **선(딜러)**: 직전 핸드 승자가 다음 판 선. 베팅은 선 다음 좌석부터

## 실행

```bash
npm install
npm run dev          # 개발 서버
npm run build        # 프로덕션 빌드
npm run test:logic   # 족보 판정 단위 테스트 + 봇 풀게임 시뮬레이션 (불변식 검증)
```

## 구조

```
src/
  game/            # 순수 게임 로직 (UI 독립 — 2단계 Firebase 재사용 대상)
    deck.ts        # 카드/덱
    handEval.ts    # 족보 판정 (베스트 5장)
    engine.ts      # 상태 머신: 드래프트 순환·베팅·사이드팟·쇼다운

    *.test.ts      # npm run test:logic
  ui/              # React 컴포넌트
  App.tsx          # 화면 라우팅 + 봇 드라이버
```

## 2단계 로드맵 (Firebase 멀티)

엔진이 `(상태, 액션) → 새 상태` 순수 함수로 설계되어 있어 그대로 이식 가능:

- 방 생성/입장 (PokéSet 룸 동기화 패턴 재사용)
- `draftPick` / `bettingAction`을 RTDB 트랜잭션으로 처리
- 턴 타이머 + 시간 초과 시 자동 폴드 (접속 끊김 = 폴드 룰)
- 칩 잔액을 기존 코인 시스템과 연동

## 멀티플레이 (Firebase) 설정

멀티플레이 버튼을 쓰려면 Firebase 프로젝트 연결이 필요합니다 (무료 Spark 플랜으로 충분).

1. https://console.firebase.google.com 에서 프로젝트 생성
2. **웹 앱 추가** (</> 아이콘) → 나오는 `firebaseConfig` 값을 `src/firebase/config.ts`에 붙여넣기
3. **Authentication > Sign-in method**에서 `이메일/비밀번호` 활성화
4. **Realtime Database** 생성 (지역: asia-southeast1 권장) → 규칙 탭에 아래 붙여넣기:

```json
{
  "rules": {
    "users": {
      ".read": "auth != null",
      ".indexOn": ["wallet"],
      "$uid": {
        ".write": "auth != null"
      }
    },
    "rooms": {
      ".read": "auth != null",
      ".write": "auth != null"
    }
  }
}
```

### 머니 시스템 (서버 단일 지갑)
- 모든 머니는 서버(RTDB `users/{uid}/wallet`) 한 곳에서 관리 — 가입 시 100만 지급
- 혼자 연습도 봇 채운 온라인 방으로 진행 (싱글/기기 지갑 없음)
- 파산(최저 채널 입장비 미만) 시 로비에서 파산 보너스 +50만 수령 가능

### 메타 시스템
- VIP 5등급 (브론즈~다이아): 경험치 = 누적 베팅액. 등급별 일일/파산 보너스 배율(×1~×4), 골드 이상 입장 이펙트
- 젬(💎): 일일 보너스·포카드(+5)·스트레이트 플러시(+20)·10연승(+10)으로 획득, 상점에서 사용
- 상점/인벤토리: 프로필 테두리(젬), 카드 뒷면 테마(골드), 이모티콘 확장팩(젬), 긴급 자금(젬→골드)
- 랭킹: 보유 골드 TOP 20 (`users`에 `.indexOn: ["wallet"]` 규칙 필요 — 위 규칙에 포함됨)

### 멀티플레이 동작 방식 (호스트 권한)
- 호스트가 엔진을 실행하고 전체 상태를 `rooms/{id}/state`에 기록, 게스트는 액션 큐로 전송
- 잠수/접속 끊김: 호스트 워치독이 베팅 18초/픽 23초에 강제 진행 (자동 다이/무작위 픽)
- 호스트가 나가면 다음 사람이 자동 승계 (상태가 DB에 있어 게임이 끊기지 않음)
- 머니 정산은 호스트가 단일 권한으로 처리: 시작 시 전원 입장비 차감 → 퇴장/게임오버 시 칩 지급
  (이를 위해 users 쓰기 규칙이 로그인 사용자에게 열려 있음 — 친구용 신뢰 모델)
- 참고: 상태에 전원의 손패가 포함되므로 개발자 도구로 상대 패를 볼 수 있음 — 친구용 v1 한계
